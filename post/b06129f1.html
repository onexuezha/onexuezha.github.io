基于Spigot搭建我的世界原版服务器

发表于2022-01-13|更新于2022-01-24|我的世界
摘要

1、安装spigot

spgot是目前最好的插件服务端

2、安装java

我的世界的依赖项

3、初始化服务器

有始有终

4、可能会遇见的bug

4.1、Unrecognized VM option ‘xxx’

4.2、Could not reserve enough space for object heap

5、正式启动服务器

恭喜

正文

安装MCSManag

打开官网速度较慢请耐心等待

按照官网的指示安装即可

安装Spigot

打开官网Spigot找到合适的版本并下载

下载完成后，你会得到一个名叫spigot-xxx.jar的文件，其中xxx是版本号

新建文件夹（最好路径中不要出现中文）将你下载到的文件拖进去，关闭文件夹

安装Java

先要清楚自己电脑是64位还是32位，并且每一个我的世界版本对应着不同的Java版本，例如1.18.1要Java17，1.12.2要Java8，具体怎么选择可以通过pcl2下载你所需要的版本，并启动下载好的游戏，如果你缺少Java的对应版本那么就pcl2会提示你下载，或者你知道每一个版本所需的java，那么自行去官网下载也可以

初始化服务器

这时打开你存放服务器核心的文件夹，在文件夹根目录下新建start.bat（注如果没有显示后缀的请在资源管理器查看界面打开）右键用记事本打开start.bat输入java -Xms1G -Xmx6G -jar xxx.jar,这里前面的1g指服务器的最小内存，后面的6g为最大内存，一般地，最大内存最好为电脑总内存÷2加2这里我的电脑总内存为8g，玩游戏最好不要开一堆其他占内存的软件，再后面的xxx.jar为你的核心名

可能会遇见的bug

这里列出了一些常见的bug，如果你遇到了其他bug欢迎反馈

Unrecognized VM option ‘xxx’

这里是你使用的开服脚本中的某个参数，不被你的Java接受

xxx可能是 +AggressiveOpts 也可能是 +UseCompressedOops 或者是你启动脚本中的其他不被接受的参数

解决办法：

将开服脚本中的包含xxx的那一段参数删除

Unrecognized VM option '+AggressiveOpts' 就删除 -XX:+AggressiveOpts

Unrecognized VM option ' +UseCompressedOops ' 就删除 -XX:+UseCompressedOops

Could not reserve enough space for object heap

设置了过大的内存

解决办法：

将开服脚本中的内存调小（注意，32位java的可用内存受限制，只能使用最大1.4G到1.6G的内存，解决办法是更换64位的Java）

将参数换成 -Xmx1G -Xms1G

内存分配一般是给系统所有可用内存大小的2/3（给系统和其他应用留出空间），如果你电脑的内存低于1G，就不要想着开服了

正式启动服务器

双击start.bat弹出命令框（如果闪退，要么电脑顶配配置一瞬间加载完，要么是你出bug了，出bug就在文件夹根目录右键运行cmd输入start里的指令，会反馈错误原因，根据提示查百度或看上面有没有相应的解决措施，没用就在评论区留言）等待执行完会自动关闭（没自动关的用stop指令关闭）此时你的文件夹应会多
出几个文件，找到eula.txt双击打开，将里面的eula项，将其从false改为true（eula.txt改为true表示你同意了使用服务器的条款，如果不开启服务器则无法启动），这时再双击一次start.bat会开始运行，恭喜，你的服务器就搭建成功了🎉🎉🎉
